# Конфигуратор КТПН — v1.0

Настольное приложение для Windows, реализующее инженерный расчёт и компоновку
КТПН на основе исходной книги `Инженерный_Расчет_КТПН_v1.0.xlsx`.
Постановка задачи — в [docs/Постановка_КТПН.md](docs/Постановка_КТПН.md).

## Готовый exe

```
build\KtpnConfigurator_V6\Конфигуратор КТПН.exe
```

Это **самодостаточная сборка** (self-contained, single-file, win-x64): .NET
устанавливать не нужно. Рядом с exe лежит папка `Data\` с редактируемыми
справочниками (трансформаторы, аппараты, списки, шаблоны документов) — её нужно
держать рядом с exe.

## Что умеет

- Каскадный выбор трансформатора (производитель → марка) с автоподстановкой
  габаритов, массы, тока НН и сечений шин из БД (73 позиции, 6 производителей).
- Конфигурация корпуса, отсеков, ввода РУВН, ввода и отходящих линий РУНН.
- Живой расчёт габаритов и масс (длина/ширина/высота, масса основания, корпуса,
  брутто) с ручным переопределением и контролем минимумов.
- Валидация проектных решений (номинал ввода ≥ ток ТМГ, диапазоны аппаратов,
  полнота заполнения) с цветовым статусом.
- Генерация документов: **Приказ на производство**, **Паспорт
  металлоконструкции**, **Чек-лист ОТК** — превью, экспорт в Excel и PDF.
- Сохранение/загрузка проекта (`*.ktpn`, JSON).

## Архитектура

| Проект | Назначение |
|--------|-----------|
| `src/KtpnConfigurator.Core` | Доменные модели, расчётное ядро (перенос формул 1:1), валидация, сборка документов, загрузка справочников. Без зависимости от UI. |
| `src/KtpnConfigurator.App` | WPF (MVVM): формы, живой пересчёт, превью, экспорт Excel (ClosedXML) / PDF (QuestPDF), сохранение проектов. |
| `tests/KtpnConfigurator.Tests` | Модульные + «золотые» тесты: результат движка сверяется с независимым оракулом (`_oracle/`) по всем ветвям формул. |
| `_oracle/` | Python-извлечение справочников из `.xlsx` и эталонные векторы (оракул). |

## Сборка из исходников

Требуется .NET SDK 10.

```powershell
# 1. (опционально) перегенерировать справочники и эталоны из исходной книги
python _oracle\extract_and_oracle.py
python _oracle\extract_docs.py

# 2. тесты
dotnet test KtpnConfigurator.slnx -c Release

# 3. самодостаточный exe
dotnet publish src\KtpnConfigurator.App\KtpnConfigurator.App.csproj -c Release -r win-x64 `
  --self-contained true -p:PublishSingleFile=true `
  -p:IncludeNativeLibrariesForSelfExtract=true -p:EnableCompressionInSingleFile=true `
  -o build\KtpnConfigurator_V6
```

## Диагностика

```powershell
.\build\KtpnConfigurator_V6\Конфигуратор КТПН.exe --selftest
```

Безголовая самопроверка: загружает справочники, считает эталонную
конфигурацию, формирует документы и экспортирует Excel/PDF во временную папку.
Код возврата `0` — всё в порядке.

## Принятые решения (по §11 постановки)

Поскольку конфигуратор v1.0 принимался без согласования открытых вопросов,
приняты следующие решения (зафиксированы в `Data/options.json`, легко
изменить):

- Номиналы ПВР/РЕ/АВ очищены до ряда `400, 630, 1000, 1600, 2000, 2500, 3150,
  4000` А.
- Номиналы ТТ: `300/5 … 2500/5`.
- Коэффициенты методики (1.15; 35 кг/м²; 1.25; ряды 2000/3000/3500; пороги
  8/4; минимум 1800 мм; порог высоты 1800 мм) вынесены в `methodology` и
  применяются как в исходной книге.
- Ручной ввод высоты не контролирует минимум (как в Excel); длина/ширина — не
  менее 1800 мм (0 трактуется как явный ноль).
- Единый источник трансформаторов — сводная БД; вендорские каталоги получаются
  фильтром по производителю.
