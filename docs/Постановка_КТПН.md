# Постановка задачи: Разработка десктопного приложения «Конфигуратор КТПН»

## 1. Введение и цели проекта
Цель проекта — миграция существующего инструмента «Инженерный расчет КТПН» (Комплектные трансформаторные подстанции наружной установки) из формата Excel-книги в полноценное десктопное приложение для ОС Windows. 

**Ключевые проблемы Excel, которые решает приложение:**
* Хрупкость логики (случайное удаление формул пользователем).
* Сложность поддержания сложного UI в рамках сетки Excel.
* Трудности с обновлением базы цен и номенклатуры комплектующих.
* Ограниченные возможности контроля ввода (валидации) данных.

**Задачи приложения:**
* Предоставить пользователю интуитивно понятный пошаговый интерфейс (Wizard) для сборки КТПН.
* Инкапсулировать и защитить расчетную логику и ценообразование.
* Реализовать надежное хранение справочников оборудования (БД).
* Автоматизировать генерацию выходной документации (спецификации, ТКП, опросные листы в формате PDF/Word/Excel).
* Позволить сохранять и загружать проекты конфигураций.

## 2. Основные функциональные блоки
1. **Модуль конфигурации (Мастер настройки):**
   * Выбор базовых параметров: Мощность (кВА), напряжение (кВ), тип исполнения (киоск, сэндвич-панели и т.д.), климатическое исполнение.
   * Конфигуратор РУВН (Распределительное устройство высшего напряжения).
   * Конфигуратор РУНН (Распределительное устройство низшего напряжения) — добавление отходящих линий, выбор автоматов.
   * Выбор трансформатора, щитов учета, собственных нужд, систем отопления и вентиляции.
2. **Модуль расчетов (Ядро):**
   * Калькуляция себестоимости и итоговой цены на основе выбранных компонентов.
   * Проверка совместимости оборудования (защита от «дурака», например, нельзя поставить автомат на 1000А в ячейку, не рассчитанную на этот ток).
   * Расчет массогабаритных характеристик (опционально, если есть в Excel).
3. **Модуль справочников (Админка):**
   * Интерфейс для добавления/редактирования/удаления комплектующих и цен (для администратора/руководителя).
4. **Модуль документов:**
   * Экспорт спецификации оборудования.
   * Формирование ТКП (Технико-коммерческого предложения).

## 3. Предлагаемая архитектура (Варианты решений)

Для разработки современного приложения под Windows я предлагаю 2 основных варианта архитектуры. Выбор зависит от наличия экспертизы у команды и предпочтений по визуалу.

### Вариант 1: Tauri + React / TypeScript (🔥 Рекомендуемый для современного UI)
Это современный подход, объединяющий веб-технологии для интерфейса и производительный системный язык для логики.
* **Frontend:** React + TypeScript + Tailwind CSS (или готовые UI-библиотеки вроде Mantine/Ant Design). Дает возможность сделать очень красивый, отзывчивый и современный интерфейс.
* **Backend (Ядро):** Rust (или Python, если использовать форки/аналоги Tauri). Rust обеспечивает высокую скорость работы, безопасность памяти и малый размер итогового файла.
* **База данных:** SQLite (локальный файл базы данных).
* **Плюсы:** Невероятно быстрый, маленький вес приложения (обычно < 10-15 МБ), интерфейс выглядит на 10/10, легко масштабируется, можно в будущем собрать под macOS/Linux.
* **Минусы:** Требует знаний React и базового понимания Rust (однако логику можно перенести полностью в JS/TS, оставив Rust только как обертку).

### Вариант 2: C# .NET 8 + WPF или WinUI 3 (🏛 Классический Enterprise)
Стандарт де-факто для инженерного софта под Windows.
* **UI-фреймворк:** WPF (Windows Presentation Foundation) или более современный WinUI 3.
* **Язык логики:** C#.
* **База данных:** SQLite (через Entity Framework Core).
* **Плюсы:** Родная среда для Windows, мощные инструменты для сложной бизнес-логики, огромная база готовых компонентов, строгая типизация, идеальная интеграция с MS Office (Word/Excel) через библиотеки (OpenXML) для выгрузки отчетов.
* **Минусы:** Интерфейс будет выглядеть чуть более «строго» и корпоративно по умолчанию; приложение работает только под Windows.

### Вариант 3: Python + PySide6 (Qt) (⚡ Для сложной математики и быстрого старта)
* **Технологии:** Python, PySide6 (Qt for Python), SQLite, Pandas.
* **Плюсы:** Если расчеты в Excel базируются на сложных математических моделях, таблицах интерполяции, то Python с библиотекой Pandas позволит перенести их быстрее всего. Огромное количество библиотек для генерации PDF/Word.
* **Минусы:** Интерфейс на Qt может выглядеть устаревшим без тщательной стилизации, большой размер сборки (`.exe` файла).

**Рекомендация архитектуры:**
Если в приоритете стильный интерфейс и "веб"-опыт — **Tauri + React**.
Если в приоритете максимальная надежность расчетов, сложная структура данных, строгий корпоративный стиль и нативная генерация Word/Excel отчетов — **C# .NET 8 (WPF/WinUI 3)**. Я бы советовал остановиться на **C#**, так как для инженерных конфигураторов это самая стабильная и проверенная экосистема.
