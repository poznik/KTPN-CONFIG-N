# Рабочий план без схем

Дата: 2026-06-19

## Решение

Схемы пока не трогаем. Не развиваем вкладку схем, УГО, листы, DXF/SVG, рендер и экспорт схем. Если понадобится, отключение схемы сделать отдельной задачей после явного решения.

Цель ближайшего этапа - довести все остальное до рабочего состояния: конфигурацию КТПН, проверки, базу оборудования, собственные нужды, документы и удобство работы.

## 1. Корпус и паспортные параметры КТПН

Сделать первым, потому что это влияет на документы и на реальную комплектацию изделия.

- климатическое исполнение и категория размещения;
- степень защиты IP корпуса, РУВН и РУНН;
- тип корпуса: металл, утепленный, сэндвич-панели;
- тип основания/рамы и примечание по фундаменту;
- заземление корпуса, PE/PEN, перемычки дверей;
- двери, замки, фиксаторы, разные ключи для РУВН/РУНН;
- вентиляционные жалюзи/решетки как паспортный параметр;
- антикоррозионное покрытие и примечания к окраске.

Результат: поля в UI, сохранение в проекте, попадание в паспорт/приказ/спецификацию, проверки полноты.

## 2. РУВН

- тип РУВН: нет, тупиковая, проходная;
- тупиковая: линия ВН приходит и питает только ТМГ;
- проходная: линия ВН приходит, питает ТМГ и уходит дальше;
- для проходной добавить три разъединителя: входящая линия, отходящая линия, ответвление на ТМГ;
- для каждого разъединителя в проходной дать возможность выбрать наличие/тип/номинал ПКТ;
- ПКТ подбирать под выбранный ТМГ по мощности и напряжению 6/10 кВ;
- тип ввода ВН: кабельный, воздушный, кабель-воздух, воздушный портал, шинный мост;
- ОПН РУВН сделать с местом установки: воздушный портал, шинный мост, защита ТМГ, не устанавливать;
- комплект кабельных принадлежностей и концевых муфт;
- проверки: нельзя выбирать ПКТ/ОПН при РУВН "нет"; для проходной показывать три разъединителя; воздушный ввод без места установки ОПН предупреждать.

## 3. РУНН и отходящие линии

Текущую простую селективность оставить:

- отходящий АВ не выше вводного аппарата;
- ТТ по номиналу автомата/ввода.

Доделать фидеры:

- имя линии;
- назначение/потребитель;
- признак резерва;
- тип вывода: кабель/воздух;
- полюсность/фазность;
- тип кабеля и сечение как справочные поля;
- комментарий по линии;
- массовые действия: копировать строку, применить модель/ТТ ко всем;
- предупреждения прямо по строкам.

## 4. Собственные нужды

- расчетная мощность освещения, розетки, обогрева, вентиляции и РИСЭ;
- итоговая мощность и ток ЩСН;
- проверка номинала вводного автомата ЩСН;
- температура включения/отключения термостата;
- тип вентиляции: естественная/принудительная;
- аварийное или ремонтное освещение;
- сервисная розетка с IP/напряжением;
- проверки: включена цепь без ЩСН, нет автомата, нет модели, нет резервируемых цепей РИСЭ.

## 5. База оборудования

- не добавлять паспортные характеристики без источника;
- добавить `sourceCheckedAt` или аналогичную дату проверки;
- показать статус `verified` / `needsVerification` в UI;
- добавить недостающие модели для ОПН, ТТ, счетчиков, светильников, розеток, обогревателей, вентиляторов, ЩСН;
- подготовить импорт/экспорт CSV позже, когда структура стабилизируется.

## 6. Проверки проекта

- группировать ошибки: критичные, предупреждения, информация;
- показывать вкладку/поле, к которому относится проблема;
- отделить "блокирует выпуск" от "проверить проектировщику";
- добавить проверки полноты по климату, IP, заземлению, дверям, РУВН, ЩСН, документам;
- добавить сводку качества проекта: сколько обязательных полей заполнено, сколько моделей требует проверки.

## 7. Документы

Фамилии не возвращать.

- опросный лист КТПН;
- ведомость комплектации;
- ведомость маркировки аппаратов, клемм и проводников;
- лист табличек и надписей;
- лист упаковки/транспортных блоков;
- протокол проверки электрических цепей/изоляции как шаблон;
- проверка плейсхолдеров шаблонов;
- все новые поля корпуса, РУВН, РУНН и ЩСН вывести в документы.

## 8. Удобство работы

- индикатор версии и даты сборки в шапке приложения;
- статус "проект сохранен / есть несохраненные изменения";
- быстрый переход из ошибки к нужной вкладке;
- компактный режим таблицы фидеров;
- понятная подсказка, какой exe запускать после сборки.

## 9. Порядок реализации

1. Корпус: климат, IP, заземление, двери, замки.
2. Документы: вывести новые паспортные поля.
3. РУНН: имя, назначение, резерв и справочные поля фидеров.
4. Проверки: группировка ошибок и привязка к вкладкам.
5. Собственные нужды: мощности, ток ЩСН, проверки цепей.
6. РУВН: тип ввода, ПКТ, ОПН, кабельные принадлежности.
7. База оборудования: статусы и недостающие группы.
8. Версия сборки и сохраненность проекта.

## 10. Чего сейчас не делать

- не трогать схему;
- не углублять селективность;
- не возвращать фамилии;
- не возвращать производителя в верхний блок количества АВ/РПС;
- не вносить паспортные характеристики оборудования без источника.
