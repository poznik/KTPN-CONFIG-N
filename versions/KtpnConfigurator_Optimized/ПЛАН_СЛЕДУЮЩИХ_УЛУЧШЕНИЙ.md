# Следующие улучшения Optimized

## 2. Автоматические и ручные значения

Цель: сразу показывать, какие параметры рассчитаны приложением, а какие изменены пользователем.

- Автоматические значения оставить с обычным фоном и подписью в подсказке.
- Ручные габариты, массы, номиналы и ТТ выделять контрастной рамкой `Задано вручную`.
- Возле каждого ручного значения добавить кнопку сброса к расчетному значению.
- В результате расчета вывести один компактный список ручных отклонений.
- В документы не добавлять служебные пояснения; печатать только итоговые значения.

## 3. Массовое заполнение РУНН

Цель: сократить повторный ввод для одинаковых отходящих линий.

- Выделение нескольких строк с общей заменой бренда, модели, номинала, учета и кабеля.
- Команда `Применить к выбранным`, не изменяющая номер и назначение линии.
- Шаблоны линий: собственные нужды, вентиляция, освещение, резерв и пользовательский шаблон.
- Копирование одной линии уже реализовано; массовое заполнение станет следующим уровнем.
- После применения сохраняются ограничения: АВ не выше ввода, АВ до 20, РПС до 8.

## 4. Сравнение версий проекта

Цель: понимать, что именно изменилось между сохранениями.

- Выбор двух файлов `.ktpn` из архива проекта.
- Сравнение по разделам: корпус, РУВН, РУНН, собственные нужды и документы.
- Показывать `было -> стало`, добавленные и удаленные отходящие линии.
- Служебные поля и порядок JSON не учитывать.
- Результат сравнения можно сохранить отдельным внутренним отчетом, но не включать в комплект документов.

## 5. Фильтрация оборудования

Цель: не предлагать заведомо неподходящие аппараты.

- Сначала фильтровать по месту установки, напряжению, типу аппарата и допустимому номиналу.
- Для отходящих АВ ограничивать список номиналом вводного аппарата.
- Для учета оставлять модели с подтвержденной совместимостью с ТТ/счетчиком.
- Рядом с моделью показывать статус: `Проверено` или `Требует проверки` и дату проверки базы.
- По умолчанию показывать все совместимые позиции; отдельный переключатель оставит только проверенные.
- Источник и статус базы остаются внутри приложения и не засоряют документы.
